### 2. 存在的 Bug 与不符预期列表（待修复）

#### 1) 首页看板菜单
- **布局高度失调**：左侧的“驾驶风险分布”图表目前高度设定较低，右侧“实时车辆位置”的地图站位却很高，两者上下高度不一致，UI 界面长短脚非常难看。
- **里程数据样式错位**：“今日里程”和“累计总里程”中的数值由于字号过大或者缺少 CSS Flex 约束，导致没能像其他指标一样，使得图标和数字在同一水平排线上。
- **预警排行榜汇总计算错误及样式**：表格“总计”列的值，不是由前面几项（驾驶、围栏、故障、低电）相加得出，数据匹配不上；同时“总计”下面的数值标签搞成了花里胡哨的红色背景（代码直接写死了 `<Tag color="red">`），没有统一成黑色常规数字。

#### 2) 车辆管理菜单
- **车辆列表详情页高度失衡**：在车辆详情页（`Vehicles.tsx`）中，左侧的“车辆信息”与“设备信息”两个独立卡片，其加起来的总高度没有实现自适应占满，未能和右侧的列表区域高度保持一致。

#### 3) 实时监控菜单
- **轨迹无法播放，功能死链**：轨迹回放中的“播放”按钮（`<Button icon={<PlayCircleOutlined />}>{t('monitor.play')}</Button>`）仅仅是一个前端空壳 UI 组件，完全没有绑定沿途经纬度动画轮播及控制逻辑，也没有按照 PRD 要求在地图下方展示动态滚动的车辆信息和速度。
- **车辆列表高度不匹配**：轨迹回放页左侧的车辆列表外层未能根据右边的主地图做伸缩并齐，高度脱节。

#### 4) 风控预警菜单
- **围栏报警筛选错误**：围栏报警列表的“报警类型”筛选下拉框中，字段值没有被正确修改为“入栏报警”、“出栏报警”和“全部”。
- **围栏名称未中文化**：围栏预警列表中，“围栏名称”字段输出的内容依然为纯英文（如 Santiago Centro Warehouse）。
- **故障报警表头及字段异常**：故障报警列表的第 3 个表头名称不明确，需要改成“报警内容”。同时，内部的字段值未能按照 PRD 做到中英文结合。
- **电池报警表头未中文化**：列表的第三个字段同样暴露出英文键值，未配置成中文。

#### 5) 驾驶报告菜单
- **查看详情事件空挂**：无论是周报还是月报列表，点击操作栏的“查看详情”按钮无效。这部分功能按钮无法正确触发针对当前周/月周期详情页的渲染。

#### 6) 电池监控菜单
- **深层翻译遗漏**：列表操作栏的表头及按钮依然是英文。详情页内部的“充电状态”直接暴露了 `charging` / `idle` 等英文枚举值，没有调整为要求的“充电中”、“空闲”等中文。

#### 7) 行程记录菜单
- **详情页布局失衡**：在查看某条行程详情时，左侧的车辆基础信息与下方的告警信息列表堆叠高度过长或过短，没有强制约束让其总高度与右侧占据主要版面的地图高度对齐。

#### 8) 围栏管理菜单
- **地名数据仍为英文**：围栏管理列表中，围栏名称和详情里的地理位置依然输出的是诸如 `Las Condes Depot`、`Av. Apoquindo 4000` 等静态测试英文单词，未按要求彻底修正为中文名和完整的中文定位。
