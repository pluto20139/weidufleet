## Spec 修订工作流（可复用）
1. 读 PRD 原文 + Review 报告 + 当前 Spec 全文
2. 按章节顺序（前→后）Edit，避免 old_string 因前面修改失效
3. 修订后全局 Grep 校验旧文案残留（防"已修订章节与未修订章节自相矛盾"）
4. 头部修订记录声称"已做"的项，必须逐项核对正文是否真的落地
5. 更新修订记录表，按 Review 编号列出，便于逐项核对
