- 采用“筛选卡片 + 数据表格”的标准布局模式，在 `Battery`, `Driving`, `Repair` 等页面中统一使用 `Card` 包裹筛选条件，下方展示 `Table` 数据。
- 广泛使用 `useMemo` 对从 Mock API 获取的数据进行缓存处理，避免在非状态变更时重复计算或请求，提升渲染性能。
- 在涉及多子视图的页面（如 `Monitor`, `Battery`, `Driving`）中，通过全局 Store 中的特定状态字段（如 `_mt`, `_bt`）控制条件渲染，而非使用 URL 参数。
- 地图组件（`MapContainer`）普遍采用固定高度容器（如 `height: 500` 或 `height: '100%'`）配合 `flex` 布局，确保在不同屏幕尺寸下的可视化效果。
