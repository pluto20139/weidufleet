# 自动化测试与 Bug 排查报告

- **审查范围**：根据 PRD 结合您在 UI 操作中指出的各项体验与功能逻辑异常，对源码（`/Users/Zhuanz1/Desktop/AI文件夹/苇渡-智利车队管理/weidu-fleet`）进行了复核，覆盖【首页看板】、【车辆管理】、【实时监控】、【风控预警】、【驾驶行为】、【电池管理】、【行程管理】、【围栏管理】、【维修管理】等全部核心模块。

### 2. 发现的问题列表 (基于 PRD 与代码排查对照)

#### 1) 首页看板菜单
- **问题位置**：`Dashboard.tsx` 及 `mock.ts`
- **错误现象**：
  1. **地图尺寸偏小**：实时位置地图区域高度写死为了 300px，页面布局比例不够大气，没有按照 PRD 合理分配版面。
  2. **数据指标样式与文案问题**：“今日里程”和“累计总里程”的单位（km）字号过大（直接使用了默认 `Statistic` 样式），没有针对前缀/后缀进行合理弱化。
  3. **今日预警排行榜数据缺失**：表格的列 `dataIndex` 定义为 `driveAlerts`、`fenceAlerts` 等，但 Mock 层返回的数据结构却是 `drive`、`fence` 等简写键名，导致页面上除车牌号与合计外，各分项预警数据全部渲染为空白。

#### 2) 车辆管理菜单
- **问题位置**：`Vehicles.tsx` 及 `mock.ts`
- **错误现象**：
  1. **车牌号格式不符**：数据依然使用的是类似 `WDLV-01`，而不是像 PRD / 用户要求中符合拉美地区风格的 `KLTX56`。
  2. **位置信息展示格式不符**：Mock 返回的仅为“Santiago”等城市名，没有按照 PRD “经纬度逆地址解析后的文字描述”展示出准确的地理位置信息（如：智利圣地亚哥首都大区圣地亚哥市阿乌马达步行街234号）。

#### 3) 实时监控菜单
- **问题位置**：实时位置 & 轨迹回放 页面 (`Monitor.tsx`)
- **错误现象**：
  1. **菜单隔离不符合 PRD**：PRD 定义中，“实时位置”与“轨迹回放”应为各自独立的二级菜单/页面。但在目前的实现中，这两个模块被平铺堆叠在了同一个页面内，导致用户下滑时会直接滑到“轨迹回放”模块。
  2. **地图白屏不可见**：`<MapContainer>` 包裹在 `<Card style={{ height: 600 }}>` 中，但 `Card` 未将高度传递给 `.ant-card-body` 容器，导致地图容器实际高度为 0 而引发白屏。

#### 4) 风控预警菜单
- **问题位置**：围栏报警、故障报警、电池报警 页面 (`Risk.tsx`)
- **错误现象**：
  1. **页面未遵循二级菜单拆分**：围栏、故障、电池三个业务在同一页面内通过垂直卡片平铺。
  2. **出栏报警空白**：`type` 字段可能因为 Mock 随机或者字段名匹配问题，在 `fenceTypeMap` 渲染时未能正确挂载。
  3. **地理位置格式不符**：列表中的“报警位置”直接输出了经纬度字符串，而不是具体的中文地理位置。
  4. **故障报警字段名与值异常**：表头由于错误拼接导致显示成“故障报警label.content”；根据 PRD 故障内容应为如“CDCU数据链路错误”等，但 Mock 仍返回英文语句。状态与操作栏展示为英文 Pending/Fixed 和 Repair，未能适配中文语境。
  5. **电池报警多语言失效**：同样，报警类型（如 Low SOC）及操作列全是英文，没有接入翻译及中文展示。
  6. **围栏报警交互未实现**：“查看”操作只绑定了 `message.info` 提示，没有弹出对应的侧边栏或详情页。

#### 5) 驾驶行为菜单
- **问题位置**：驾驶预警、驾驶报告 页面 (`Driving.tsx`)
- **错误现象**：
  1. **生造 Tab 栏**：“驾驶预警”与“驾驶报告”为并列二级路由，被做成了多余的内部 Tabs 切换。
  2. **周报/月报无联动差异**：在“驾驶报告”的 Tab 中，周报与月报虽然能够切换 Tab，但下方引用的 Table 和请求的数据源完全是静态的（`drivingReports`），没有随着周报/月报的切换改变其呈现的周期（Period）。
  3. **字段取值与中文化遗漏**：预警类型的字段（Rapid Accel 等）、位置信息（经纬度而非中文地理位置）、风险等级（Safe 等）全是英文字符串。
  4. **表头语义及键名错用**：操作栏表头漏翻为 `common.action`；车速表头展示为了“km/h”（应为“车速”）；图表和详情页中的键名多处缺失翻译，直接暴露出 `driving.mileage` 等。

#### 6) 电池管理菜单
- **问题位置**：电池监控、充放电记录 页面 (`Battery.tsx`)
- **错误现象**：
  1. **中文化漏翻**：电池监控中的“充电状态”、数据指标中第4个统计卡片以及表格中的操作栏等均没有实现中文化，显示为英文。
  2. **层级错误与冗余**：“充放电记录”页面中的 Tab 栏错误地包含了“电池监控”页签。
  3. **放电记录表头文字错误**：PRD 要求放电记录应展示“放电前/放电后”，但复用了充电列表导致展示为“充电前/充电后”。

#### 7) 行程管理菜单
- **问题位置**：行程记录 页面 (`Trips.tsx`)
- **错误现象**：
  1. **详情页布局失衡**：在详情模式下，左侧的 `Descriptions` （车辆行程信息）占据了 14 栅格，右侧 10 栅格塞入了地图和告警信息列表。按需求，告警信息和行程信息应同置于左侧，地图应当独占右侧并适当放大。

#### 8) 围栏管理菜单
- **问题位置**：围栏管理 页面 (`Fence.tsx`)
- **错误现象**：
  1. **状态开关文字冗余**：列表“状态”列展示 `Switch` 时携带了冗余文字。
  2. **地图无法选点交互**：在“新建围栏”的弹窗中，虽然提示了在地图上标记范围，但并没有实现 `<MapContainer>` 的点击获取坐标(`useMapEvents`)功能，无法通过交互获取中心点或自定义坐标。
  3. **编辑功能挂空**：列表操作栏的编辑按钮未绑定任何事件，无法进入编辑态。操作列表头也漏翻为 `common.action`。

#### 9) 维修管理菜单
- **问题位置**：维修记录 页面 (`Repair.tsx`)
- **错误现象**：
  1. **表头未翻译**：操作列显示为 `common.action`。
  2. **Mock 数据为英文**：列表“描述”在中文语言下，仍展示了大量英文 Mock 数据（如 "Motor controller communication error..."），未能正确体现中文业务语境。
