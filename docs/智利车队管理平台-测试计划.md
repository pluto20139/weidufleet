# 智利车队管理平台 - 测试计划

## 1. 项目背景
本项目为“苇渡-智利车队管理平台”，旨在为智利境内的物流车队提供全方位的车辆管理、监控、预警及运营支持。系统采用 React + Ant Design + Leaflet 地图技术构建。

## 2. 测试目标
* 验证系统所有功能模块是否符合 PRD 要求。
* 确保多语言（中、英、西）切换正常且无翻译缺失。
* 验证车辆实时监控及轨迹回放的准确性。
* 确保系统在不同分辨率及主流浏览器下的兼容性。
* 验证租户权限隔离的安全性。

## 3. 测试范围
### 3.1 功能测试
* **基础能力**：登录/登出、修改密码、语言切换。
* **首页看板**：统计指标、今日预警、车辆分布地图。
* **车辆管理**：车辆搜索/重置、列表分页、详情页（多维度数据展示）。
* **实时监控**：车辆位置展示、轨迹回放。
* **风险预警**：围栏报警、故障报警、电池报警。
* **驾驶行为**：驾驶记录预警、周报/月报生成。
* **围栏管理**：围栏创建、删除、地图查看。
* **管理模块**：租户管理、业务权限分配、用户与角色管理。

### 3.2 专项测试
* **国际化 (i18n)**：检查所有静态文本及动态报错信息的语言正确性。
* **兼容性**：Chrome, Safari, Edge 浏览器及不同屏幕尺寸（字号调大后的适配）。
* **性能**：大量车辆点位的地图渲染流畅度。

## 4. 测试策略
* **单元测试**：针对核心逻辑（如数据转换函数、权限判断逻辑）使用 Jest 进行测试。
* **集成测试**：验证各功能组件（如 Sidebar 菜单与状态库 Zustand 的联动）的交互。
* **黑盒测试/UI 测试**：由测试人员模拟真实用户操作，覆盖 PRD 功能点。
* **回归测试**：在代码修复（如字号调整、详情页报错修复）后进行全量回归。

## 5. 资源要求
* **硬件**：MacOS/Windows PC。
* **工具**：Chrome DevTools, Postman (接口测试)。
* **环境**：本地开发环境或测试服务器环境。

## 6. 测试进度安排
* **第一阶段**：冒烟测试（登录及核心模块），预计 1 天。
* **第二阶段**：功能全覆盖测试（含 i18n），预计 3 天。
* **第三阶段**：缺陷修复验证及回归测试，预计 2 天。

## 7. 风险点
* **地图加载**：海外服务器（智利）环境下 OpenStreetMap 的访问速度。
* **数据准确性**：Mock 数据与真实 API 接入后的逻辑一致性。
* **权限覆盖**：多租户及复杂角色权限组合下的越权访问风险。
