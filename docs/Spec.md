## 3. 全局架构设计

### 3.1 路由表

| 路径 | 页面组件 | 说明 |
|---|---|---|

| `/risk` | Risk | 风控预警（内部tab: fence/fault/battery） |
| `/driving` | Driving | 驾驶行为（内部tab: alert/report） |
| `/battery` | Battery | 电池管理（内部tab: monitor/charge） |

| `/sys/roles` | Sys | 系统管理-角色管理 |
| `*` | → Navigate `/dashboard` | 未知路径重定向 |

**路由实现要点**：
- 所有页面组件通过 `React.lazy()` 懒加载
- `AppLayout` 组件包裹所有受保护路由，提供 Sidebar + Topbar + Content 布局
- 认证检查：`useAppStore.page === 'login'` 时重定向到 `/login`
- 内部tab切换通过 Zustand store 持久化（`_rt`, `_dt`, `_bt`, `_mt`, `bz` 等），URL query param 仅用于侧边栏菜单定位

### 3.2 状态管理 (Zustand)

`useAppStore` 使用 `persist` 中间件，`localStorage` key 为 `weidu-fleet-storage`。

| 状态字段 | 类型 | 持久化 | 说明 |
|---|---|---|---|
| `page` | `PageKey \| 'login'` | **是** | 当前页面标识（认证态） |
| `user` | `{ name, email, mustChangePwd? }` | **是** | 登录用户信息 |
| `token` | `string` | **是** | 认证令牌 |
| `lang` | `'zh' \| 'en' \| 'es'` | **是** | 当前语言 |
| `tenant` | `string` | **是** | 当前选中租户 |
| `tenants` | `TenantItem[]` | 否 | 可切换租户列表 |
| `detail` | `string` | 否 | 当前查看详情的VIN |
| `_vf` | `{ vin?, plate?, device?, batteryVer?, minAge?, maxAge? }` | 否 | 车辆列表筛选条件 |
| `_rt` | `string` (default: `'fence'`) | 否 | 风控预警当前tab |
| `_dt` | `string` (default: `'alert'`) | 否 | 驾驶行为当前tab |
| `_dr` | `string` (default: `'week'`) | 否 | 驾驶报告周期 |
| `_bt` | `string` (default: `'monitor'`) | 否 | 电池管理当前tab |

| `bz` | `string` (default: `'permission'`) | 否 | 业务管理当前tab |

## 4. 全局规则

### 4.1 时区处理

- 所有时间统一转换为智利时区 `America/Santiago`
- 使用 `dayjs` 的 `utc` + `timezone` 插件
- 夏令时 (UTC-3): 每年9月第一个星期六 24:00 ~ 次年4月第一个星期六 24:00
- 冬令时 (UTC-4): 每年4月第一个星期六 24:00 ~ 9月第一个星期六 24:00
- 标准格式: `YYYY-MM-DD HH:mm:ss`
- 工具函数: `formatTime(date)` → `dayjs(date).tz().format('YYYY-MM-DD HH:mm:ss')`

### 4.2 地理位置显示精度

| 规则 | 说明 |
|---|---|
| 精度限制 | 逆地址解析最高保留至街道/路口级别，移除门牌号/建筑号 |
| 中文地址 | 移除末尾"数字+号"，如 `步行街234号` → `步行街` |
| 西文地址 | 移除末尾数字，如 `Av. Libertador 1500` → `Av. Libertador` |
| 实现方式 | `LocationPrivacy` 组件自动截断 + 直接展示处手动调用截断函数 |
| 适用范围 | 实时监控气泡、轨迹行程起终点、风控报警位置、行程起终点、车辆详情位置 |

## 6. 功能模块规格

### 6.1 登录及基础能力

**文件**: `src/pages/Login.tsx`, `src/components/Layout/Topbar.tsx`

#### 登录
- 输入字段: 邮箱(账号)、密码、验证码
- 登录流程: 调用登录接口 → 存储 token + user 到 Zustand → 跳转 `/dashboard`
- 异常提示:
  - 账号密码错误 / 验证码错误

#### 修改密码
- 场景1: 新用户或重置过密码的用户，登录后自动弹出修改密码弹窗（`mustChangePassword` 标志）
- 场景2: 用户通过 Topbar 右上角主动发起
- 字段: 当前密码、新密码(8~18位, 数字+英文+特殊字符)、确认新密码
- 校验: 当前密码正确性、新密码格式、两次一致性

#### 语言切换
- Topbar 提供 zh / en / es 三语切换
- 切换后调用 `i18n.changeLanguage()` + `setLang()` 持久化
- 默认语言: zh

#### 租户切换
- Topbar 展示当前租户名称，下拉显示可切换的租户列表（平铺展示，按关联时间排序，最新在最下方）
- 仅关联一个租户时隐藏切换按钮
- 记忆最近一次选择的租户；若最近一次选中的租户已到期，则默认选中下一个未到期的租户
- 选中后刷新菜单（根据该租户下的角色权限展示）
- 若登录账户关联的一级租户服务到期，则该一级租户及二三级租户显示"服务过期"

### 6.2 首页看板

**文件**: `src/pages/Dashboard.tsx`

#### 数据指标卡片 (8项)
| 指标 | 数据来源 | 说明 |
|---|---|---|
| 车辆总数 | 平台所有车辆 | `totalVehicles` |
| 在线数 | OBD设备在线 | `online` |
| 离线数 | OBD设备离线 | `offline` |
| 今日里程 | 所有车辆今日里程 | `todayMileage` |
| 累计总里程 | 历史里程总数 | `totalMileage` |
| 今日驾驶预警 | 今日驾驶预警总数 | `todayAlerts` |
| 今日围栏预警 | 今日围栏预警总数 | `fenceAlerts` |
| 今日故障预警 | 今日故障预警总数 | `faultAlerts` |
| 今日低电报警 | SOC过低的电池报警数 | `lowBatteryAlerts` |

#### 驾驶风险分布图
- 图表类型: 柱状图 (chart.js Bar)
- 数据维度: 急加速、急减速、急转弯、疲劳驾驶、AEB制动
- 计算方式: 每类事件数量的占比
- 筛选: 今日 / 近7日 / 近30日（默认今日）

#### 实时位置地图
- 地图组件: react-leaflet MapContainer
- 展示所有车辆位置打点 (CircleMarker)，车辆数量根据地图缩放做聚合(cluster)处理
- 地图比例拉到最大时，定时分批刷新车辆数据
- 点击展示 Popup: 车牌号、VIN、企业名称、设备名称、设备状态、电池信息(SOC)、行驶速度、上报时间
- 地图支持缩放、拖动

#### 今日预警排行榜
- 列表字段: 排名、车牌号、驾驶预警次数、围栏预警次数、故障预警次数、低电报警次数、总计
- 排序: 按总计由高到低
- 取前8辆


### 6.3 车辆管理

**文件**: `src/pages/Vehicles.tsx`, `src/pages/Vehicles/*.tsx`

#### 车辆列表
- **数据权限**: 当前租户及下级租户名下所有车辆
- **列表字段**:
  - VIN码
  - 车牌号
  - 绑定设备ID（车辆绑定的OBD设备ID）
  - 车型
  - 外观（颜色）
  - 电池版本
  - 购车时间 (YYYY-MM-DD)
  - 车龄（动态计算: `(now - purchase) / 365`, 保留1位小数）
  - 总里程 (km, 整数)
  - 最后位置（过长省略+气泡展示完整）
- **搜索筛选**:

| 字段 | 输入方式 | 限制 | 匹配 |
|---|---|---|---|
| VIN码 | 文本框 | ≤17字符, 字母+数字 | 模糊(matchVinSearch) |
| 车牌号 | 文本框 | ≤8字符, 汉字+字母+数字 | 模糊(matchPlateSearch) |
| 设备ID | 文本框 | ≤20字符, 字母+数字 | 模糊 |
| 电池版本 | 文本框 | ≤20字符, 字母+数字+中文 | 模糊 |
| 车龄 | 数值范围 | 0~100, 1位小数 | 范围命中 |

- **操作**: 批量导入、查看详情
- **分页**: 默认20条, 支持 10/20/50/100
- **排序**: 按车辆添加时间倒序

#### 批量导入
- 前提: VIN必须已存在于当前租户资产池
- 可覆盖字段: 外观、车牌号
- 不可覆盖: VIN、购车时间、车型、类型、绑定设备号、车龄、总里程、电池版本、最后位置
- 下载模板: xlsx格式
- 文件校验:
  - 格式错误 → 整体失败
  - 文件>50MB → 整体失败
  - VIN为空/格式错/重复/不存在 → 跳过该条，生成失败明细
- 失败明细: 记录原始错误数据，支持excel下载

#### 车辆详情 (`/vehicles/:vin`)
- **左侧**: 车辆信息 + 设备信息
  - 车辆: VIN、车牌、车型、外观、**类型**、购车时间、车龄、电池版本、总里程、最后位置
  - 设备: 设备ID、设备名称、设备类型、设备型号
- **右侧**: Tab列表（默认选中风控预警记录）

| Tab | 子组件 | 列表字段 |
|---|---|---|
| 风控预警记录 | AlertTable | **VIN码、车牌号、**预警名称(23种)、预警内容(CAN解码)、预警时间 |
| 驾驶预警记录 | DrivingTable | **VIN码、车牌号、**预警名称(6种)、**预警内容(CAN解码)、**预警时间 |
| 电池监控信息 | BatteryTable | **VIN码、车牌号、**SOC、电池健康度、电池温度、续航里程 |
| 充电记录 | ChargeTable | **VIN码、车牌号、**电压(V)、电流(A)、功率(kW)、充电前电量(%)、充电后电量(%)、充电时长、记录时间(YYYY-MM-DD HH:mm:ss) |
| 行程记录 | TripTable | **VIN码、车牌号、**起止时间、起终点(位置隐私)、行驶里程、行程时长、平均车速、预警次数(驾驶+围栏+故障+低电) |
| 维修记录 | RepairTable | **VIN码、车牌号、**维修类型、维修描述、开始时间、结束时间、记录人、维修状态、完成维修按钮 |
| 里程报表 | MileageChart | 折线图; X轴: 日(近30天)/周(近30周)/月(近12个月)/年(近6年); Y轴: km |

- 风控预警名称枚举(23种): VDC故障、CDCU故障、BDCU故障、ADAS故障、温度差异、电池高温、车载储能装置过压、车载储能装置欠压、SOC低、单体电池过压、单体电池欠压、SOC过高、SOC跳变、可充电储能系统不匹配、电池单体一致性差、绝缘、DC-DC温度、DC-DC状态、驱动电机控制器温度、高压互锁状态、驱动电机温度、车载储能装置类型过充、充电故障

| 次数 | 急加速 | 急减速 | 急转弯 | AEB制动 | 疲劳驾驶(时长h) |

| 0 | 100 | 100 | 100 | 100 | 0h=100 |
| 2 | 90 | 90 | 90 | 90 | 1h=90 |
| 4 | 80 | 80 | 80 | 80 | 2h=80 |
| 6 | 70 | 70 | 70 | 70 | 3h=70 |
| 7-10 | 60 | 60 | 60 | 60 | 4h=60 |
| >10 | 50 | 50 | 50 | 50 | ≥5h=50 |

| 指标 | 判定标准 |
|---|---|

- **编辑用户**: 回显信息 → 邮箱唯一性校验(已被使用则输入框下飘红) → 保存; **管理员角色用户角色置灰不可更改**
- **删除用户**: 二次确认 → 仅删除该用户在该租户下的角色权限关系(其他租户不受影响); **管理员角色账号不可删除**
- **搜索**: 昵称(模糊,长度不限) + 邮箱(模糊,长度不限) + 角色(多选下拉,模糊) + 创建时间(区间)
- 分页: 默认10条, 10/20/50/100, 按创建时间倒序

#### 角色管理 (tab: roles)
- 左侧租户树 + 右侧角色列表
- 角色按创建时间正序(最新在最下方)
- **管理员角色**: 不可编辑/删除/修改功能权限/修改数据权限; 鼠标移入时编辑/删除图标不展示
- **新增角色**: 名称(≤10字符, 与租户下已有角色唯一校验, 重复飘红"角色已存在") + 功能权限编辑(范围: 该租户层级下可配置权限) + 数据权限(默认全部下级租户)
- **删除角色**: 二次确认弹窗
  - 未关联用户: "删除后数据无法恢复，是否继续？"
  - 已关联用户: "该角色已关联用户，删除后用户无相关权限，是否继续？"
- 鼠标移入角色显示编辑/删除图标; 点击角色右侧展示功能权限和数据权限

### 6.13 系统管理

**文件**: `src/pages/Sys.tsx`

#### 用户管理 (tab: users)
- 根据登录用户所在租户层级，查询该租户下创建的用户信息
- 字段: 序号、用户昵称、登录邮箱、角色(多个用"、"分隔)、创建时间、操作(重置密码/编辑/删除)
- **新增用户**: 两步创建流程(同业务管理); **选择角色时，管理员角色置灰不可选择**
- **重置密码**: 二次确认 → 重置 → 弹窗展示(昵称/邮箱/新密码) → 复制 → Toast"复制成功"
- **编辑用户**: 回显 → 邮箱唯一性校验(飘红) → 保存; **管理员角色用户角色框置灰不可修改**
- **删除用户**: 二次确认 → 仅删除在该租户下的角色权限关系; **管理员角色账号不可删除**
- **搜索**: 昵称(模糊) + 邮箱(模糊) + 角色(多选下拉,模糊) + 创建时间(区间)
- 分页: 默认10条, 10/20/50/100, 按创建时间倒序

#### 角色管理 (tab: roles)
- 根据登录用户所在租户层级查询角色信息
- 角色按创建时间正序(最新在最下方)
- **管理员角色**: 不可删除/不可配置功能权限/**仅可查看**
- **新增角色**: 名称(≤10字符,唯一校验) + 功能权限编辑 + 数据权限(默认全部下级)

#### 管理员角色同步规则(租户创建/开通账号触发):
1. 租户创建后 → 同步**管理员角色**到该租户的角色管理中
2. 租户管理中设置管理员账号时 → 角色默认固定为管理员，不可选
3. 租户设置管理员账号后 → 同步该**管理员账号**到该租户的用户管理中
4. 角色管理中 → 管理员角色不可删除/不可配置功能权限/仅可查看
5. 用户管理中 → 同步过来的管理员账号不可删除; 编辑时角色框置灰
6. 用户管理中新增其他用户时 → 管理员角色在选择列表中置灰不可选

## 8. 状态流转

### 8.1 报警状态(故障报警/电池报警)

```
[未处理] ──一键报修──▶ [已生成工单] ──维修完成──▶ [已修复]
```

- "未处理" → "已生成工单": 点击一键报修按钮(仅未处理时展示)
- "已生成工单" → "已修复": 关联维修工单完成维修时同步更新

### 8.2 维修状态

```
[维修中] ──完成维修──▶ [已完成]
```

- "维修中": 新建维修/一键报修生成的初始状态
- "已完成": 点击"完成维修"按钮(仅维修中时展示), 记录结束时间
- **映射关系**: 故障/电池报警"一键报修" → 生成"维修中"维修记录 + 报警状态变为"已生成工单"；维修记录"完成维修" → 报警状态同步变为"已修复"

### 8.3 围栏状态

```
[生效中] ──关闭(二次确认)──▶ [未生效]
[未生效] ──开启───────────▶ [生效中]
```

- 删除: 仅"未生效"状态可删除(二次确认)
- 编辑: 任何状态均可编辑

### 8.4 租户状态

| 状态 | 触发条件 | 影响 |
|---|---|---|
| 正常 | 默认 | 正常访问 |
| 服务过期 | 一级租户服务到期 | 该一级及所有二三级租户显示"服务过期"，菜单和数据受限 |
