- **审查范围**：根据 PRD 结合您在 UI 操作中指出的各项体验与功能逻辑异常，对源码（`/Users/Zhuanz1/Desktop/AI文件夹/苇渡-智利车队管理/weidu-fleet`）进行了复核，覆盖【实时监控】、【风控预警】、【驾驶行为】、【电池管理】、【行程管理】、【围栏管理】、【维修管理】等核心模块。

### 2. 发现的问题列表 (基于 PRD 与代码排查对照)

#### 1) 实时监控菜单
- **问题位置**：实时位置 & 轨迹回放 页面 (`Monitor.tsx`)
- **错误现象**：
  1. **菜单隔离不符合 PRD**：PRD 定义中，“实时位置”与“轨迹回放”应为各自独立的二级菜单/页面。但在目前的实现中，这两个模块被平铺堆叠在了同一个页面内，导致用户下滑时会直接滑到“轨迹回放”模块，这是结构规划的错误。
  2. **地图白屏不可见**：地图区域加载失败呈现空白。经审查代码发现，`<MapContainer>` 被包裹在 Ant Design 的 `<Card style={{ height: 600 }}>` 中。但 `Card` 默认并未将高度传递给其内部的 `.ant-card-body` 容器，导致包裹地图的直接父节点高度为 0，从而引发白屏。

#### 2) 风控预警菜单
- **问题位置**：围栏报警、故障报警、电池报警 页面 (`Risk.tsx`)
- **错误现象**：
  1. **页面未遵循二级菜单拆分**：与上述类似，违背 PRD 二级菜单设计。代码将围栏、故障、电池三个业务的表格在同一页面内通过垂直 `<Card>` 直接铺开。
  2. **表头与字段翻译失效**：操作列的表头直接暴露了原始语种键 `common.action`；故障/电池报警的报警内容列错误拼接了键名（代码里写成了 `t('risk.fault') + t('label.content')`，表现为“故障报警label.content”）。
  3. **业务数据硬编码为英文**：故障报警/电池报警中的处理状态为硬编码的 Pending / Fixed / WorkOrder 等英文状态；由于 `fenceTypeMap` 数据源错位，出栏报警处于空白看不见的状态。
  4. **交互未实现**：围栏报警的“查看”操作只绑定了 `message.info` 提示气泡（`View fence alert: fa-0`），并没有按常规功能弹出抽屉或详情 Modal。

#### 3) 驾驶行为菜单
- **问题位置**：驾驶预警、驾驶报告 页面 (`Driving.tsx`)
- **错误现象**：
  1. **生造 Tab 栏**：PRD 中明确“驾驶预警”与“驾驶报告”为并列的二级路由，但开发时被做成了多余的内部 Tabs 切换。
  2. **字段取值与中文化遗漏**：预警类型的字段（Rapid Accel / Hard Brake）、列表中的风险等级（Safe / Low / Medium / High）均是英文字符串。定位信息也没有调用逆解析地址而是经纬度/坐标点形式。
  3. **表头语义及键名错用**：操作栏表头漏翻为 `common.action`；车速被定名为单位“km/h”（应为“车速”）；“查看详情”按钮的键名被错误套用了围栏的 `fence.detail`。
  4. **详情页多语言失效**：部分图表及展示位的键名如 `driving.mileage`，`driving.risk_events` 未挂载有效翻译，直接暴露为英文字符。

#### 4) 电池管理菜单
- **问题位置**：电池监控、充放电记录 页面 (`Battery.tsx`)
- **错误现象**：
  1. **层级错误与翻译遗漏**：同上存在 `common.action` 漏翻；“充放电记录”页面中的 Tab 栏错误地包含了“电池监控”（应剔除）。
  2. **放电记录文案套用错误**：PRD 要求放电记录中应展示为“放电前/放电后”，但代码中由于字段与充电复用，直接展示为了“充电前/充电后”。

#### 5) 行程管理菜单
- **问题位置**：行程记录 页面 (`Trips.tsx`)
- **错误现象**：
  1. **详情页布局失衡**：在详情模式下，左侧的 `Descriptions` （车辆行程信息）占据了过大版面（14栅格），右侧较小的空间（10栅格）却同时塞入了地图和告警信息列表。根据需求，告警信息和行程信息应同置于左侧，地图应当独占右侧并扩大占位面积。

#### 6) 围栏管理菜单
- **问题位置**：围栏管理 页面 (`Fence.tsx`)
- **错误现象**：
  1. **状态列文案冗余**：列表“状态”列展示 `Switch` 开关时，携带了不需要的“生效”文字（`checkedChildren`）。
  2. **编辑功能缺失**：操作栏显示为 `common.action`。另外，代码中的编辑按钮 `<Button type="link">{t('fence.action.edit')}</Button>` 完全是一个没有绑定任何 `onClick` 事件的空壳，导致无法进入编辑态。

#### 7) 维修管理菜单
- **问题位置**：维修记录 页面 (`Repair.tsx`)
- **错误现象**：
  1. **表头未翻译**：操作列同样显示为 `common.action`。
  2. **Mock 数据非中文**：在中文语言环境下，列表“描述”内容由于 API Mock 数据 (`src/api/mock.ts` 中 `repairData`) 写死了英文长句（例如 "Motor controller communication error..."），导致展现全是英文。未能模拟中文语境。
