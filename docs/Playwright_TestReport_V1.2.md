#### 失败原因分析

| TC | 失败类型 | 详细原因 |
|----|----------|----------|
| **TC-S1-03** | 脱敏未实现 | VIN列显示明文 `"LJ8T7AD100000"`，未被脱敏为 `LJ8T7A*******0000` 格式 |
| **TC-S1-04** | 搜索后无数据 | VIN搜索后表格行被隐藏（`aria-hidden="true"`），table显示measure row替代数据行 |
| **TC-S1-05** | 无"查看"链接 | 车辆列表页无法找到 `table a` 中含"查看"文本的链接，详情页入口不存在或文本不匹配 |
| **TC-S1-06** | 同上 | 依赖车辆详情页入口（先点"查看"进入详情），入口不存在导致级联失败 |
| **TC-S1-07** | 车牌格式异常 | 脱敏后的车牌包含 `"—"` 占位符，不符合 `前2+**+后2` 的脱敏格式要求 |
| **TC-S1-09** | Modal内容缺失 | 围栏报警详情Modal中未找到脱敏VIN格式文本 `LJ8T7A*******` |
| **TC-S1-10** | 页面无表格 | `/driving?tab=alert` 页面未渲染 `.ant-table`，页面路由或组件加载异常 |
| **TC-S1-12** | 脱敏格式错误 | 车牌显示为 `"京A****62"`（4个星号），期望格式为 `"京A**62"`（2个星号）|
| **TC-S1-16** | 选择器冲突 | `headers.toContainText('车牌号')` 触发了 strict mode 冲突（4个th元素匹配），需改用 `filter` 或遍历 |

#### 关键发现

1. **VIN脱敏未全局实施**：车辆列表页（`/vehicles`）的VIN列仍显示明文
2. **车牌脱敏星号数量不一致**：行程管理页使用 `****`（4星），而排行榜/电池管理等页使用 `**`（2星）
3. **车辆列表"查看"链接选择器不匹配**：实际UI可能使用按钮或其他元素，需检查实际DOM结构
4. **驾驶预警页面无数据**：`/driving?tab=alert` 未渲染表格，可能路由参数或组件状态有误

#### 🟡 P1 — UI/交互缺陷（功能可用但不完整）

| # | 问题 | 影响TC | 描述 |
|---|------|--------|------|
| P1-1 | "查看"链接选择器不匹配 | S1-05, S1-06, S3-02 | 车辆列表/行程列表的详情入口可能使用按钮或图标，非 `<a>` 标签 |

|---|------|--------|------|
| P2-1 | toContainText strict mode 冲突 | S1-16 | 多元素匹配时应用 `filter({ hasText })` 替代直接 `toContainText` |

#### 脱敏模块
- **已实现**: 排行榜、地图气泡、风控预警、电池管理、维修管理、实时监控

- **缺失**: "操作结果"筛选组件、数据行可见性问题
- **建议**: 将表格行选择器统一改为 `tr.ant-table-row`

#### 轨迹模块
- **未实现**: 轨迹回放日期选择器、10天限制、超期遮罩
- **依赖**: 行程详情入口问题（与脱敏模块共享）

#### 地址精度模块
- **正常**: 车辆列表最后位置、行程列表起终点
- **异常**: 驾驶预警页面（与脱敏模块共享路由问题）
