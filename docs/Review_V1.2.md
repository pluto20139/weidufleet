## 2. 功能缺陷（真 Bug）

### BUG-1: 车牌脱敏星号数量与 PRD 不一致

| 项目 | 内容 |
|------|------|
| **PRD 规定** | 首尾各保留 2 位，中间 `*` 替换 → `ABCD12 → AB**12`（固定 2 星） |
| **代码实现** | `'*'.repeat(plate.length - 4)` → 7 位车牌 `京A88862` → `京A***62`（3 星） |
| **实际结果** | TC-S1-12 检测到 `京A****62`（4 星？需确认 mock 数据实际位数） |
| **影响** | 行程管理页车牌显示格式与排行榜、电池管理等页面不一致 |
| **修正** | PRD 的"中间"是固定 2 颗星还是动态长度？需要和 PM 确认。当前代码对 6 位车牌 `ABCD12` 正确（`AB**12`），但对 7 位/8 位车牌产生多星。 |

### BUG-2: 围栏配置页车牌列含 `—` 占位符

| 项目 | 内容 |
|------|------|
| **PRD** | 围栏管理的 VIN/车牌全部脱敏显示 |
| **代码** | 未绑定车辆时，表格列显示 `—`，脱敏校验将 `—` 判为格式错误 |
| **影响** | TC-S1-07 失败——占位符不属于脱敏格式，但它是预期的业务占位符 |
| **修正** | 测试应在断言前跳过值为 `—` 的单元格；业务逻辑无需修改 |

## 3. PRD/Spec 歧义需确认

### AMBIGUITY-1: 车牌脱敏"中间"星号数量

**PRD 原文**: "首尾各保留2位，中间使用星号*替换的策略"

**Spec 实现**: `'*'.repeat(plate.length - 4)` 

**歧义**: 
- 若"中间"指固定 2 颗星：6 位 `AB**12`，7 位 `京A**62`，但 7 位车牌保留首尾 2 位后中间剩 3 位需要覆盖
- 若"中间"指保留首尾后剩余所有位：6 位 `AB**12`，7 位 `京A***62`

**建议**: 和 PM 确认 7~8 位车牌的脱敏格式。当前代码的逻辑更合理（保留总长度-4 颗星），因为它保持了原始车牌长度不变。

## 4. 测试脚本问题（13 项）

### 4.1 路由导航方式与 app 架构不匹配（影响 4 项测试）

| 失败 TC | 问题 | 原因 |
|---------|------|------|
| TC-S1-10 | 驾驶预警页无表格 | 测试直接 `goto('/driving?tab=alert')`，app 的 `_dt` store 未更新，页面的 `{_dt === 'alert' && alertTab}` 条件不成立，表格不渲染 |
| TC-S2-06 | 同上 | 同上 |
| TC-S3-01 | 轨迹回放日期选择器 | 测试直接 `goto('/monitor?tab=playback')`，store `_mt` 未更新，`{showPlayback && trajectorySection}` 条件不成立 |
| TC-S1-08 | 风控预警 | （通过）但同样直接 goto，运气好因为它重新渲染了？ |

**修复**: 对于这类依赖 store 的页面，测试需要通过侧边栏点击导航，或先 `page.evaluate(() => setRt('alert'))` 再 goto。建议测试计划统一加一条 `page.evaluate` 来设置 store：

```ts
// 在 goto 之前设置 store tab 状态
await page.evaluate(() => {
  localStorage.setItem('weidu-fleet-storage', JSON.stringify({
    state: { _dt: 'alert', user: {...}, token: '...', lang: 'zh' }, version: 0
  }));
});
```

### 4.2 详情入口选择器类型错误（影响 4 项测试）

| 失败 TC | 问题 |
|---------|------|
| TC-S1-05 | 车辆详情 — 使用了 `table a filter hasText 查看`，但实际渲染是 `<Button type="link">`（antd 5.x 渲染为 `<button>` 非 `<a>`） |
| TC-S1-06 | 同上 |
| TC-S3-02 | 同上 |
| TC-S1-09 | 围栏报警详情 — 同上，Modal 中搜索 `LJ8T7A*******` 但实际 mask 格式后包含后 4 位 `0000` |

**修复**: 
- 将 `page.locator('table a').filter({ hasText: /查看/ })` 改为 `page.getByRole('button', { name: /查看/ })` 或 `page.locator('button').filter({ hasText: /查看详情/ })`
- TC-S1-09 将正则改为匹配完整 17 位格式 `LJ8T7A\*{7}[A-Z0-9]{4}`

### 4.3 antd Table 行选择器不精确（影响 3 项测试）

| 失败 TC | 问题 |
|---------|------|
| TC-S4-03 | `.ant-table-tbody tr` 首选匹配到 `aria-hidden="true"` 的 measure row |
| TC-S4-07 | 同上 |
| TC-S1-04 | 搜索后无法获取数据行——同 measure row 问题 |

**修复**: 统一改用 `tr.ant-table-row` 或 `tr[data-row-key]` 定位数据行：

```ts
// 错误
const rows = page.locator('.ant-table-tbody tr');
// 正确
const rows = page.locator('.ant-table-tbody tr.ant-table-row');
```

### 4.4 toContainText strict mode（影响 1 项测试）

| 失败 TC | 问题 |
|---------|------|
| TC-S1-16 | `headers.toContainText('车牌号')` 匹配到 4 个 `<th>` 元素，strict mode 报错 |

**修复**: 使用 `filter({ hasText: '车牌号' })` 或遍历：

```ts
await expect(page.locator('th').filter({ hasText: '车牌号' })).toBeVisible();
```

### 4.5 脱敏断言格式不统一（影响 1 项测试）

| 失败 TC | 问题 |
|---------|------|
| TC-S1-07 | 围栏列表车牌列含 `—` 占位符，脱敏正则校验失败。不是 Bug，是测试未处理占位符 |

**修复**: 在 `expectMaskedPlate` 中增加 `—` 白名单：

```ts
export function expectMaskedPlate(text: string): boolean {
  if (text === '—') return true; // 占位符跳过
  return /^.{2}\*{2}.{2}$/.test(text);
}
```

## 7. 结论

### 必须修复项

1. **驾驶预警、轨迹回放、风控预警等页面的 store 状态问题** → 测试导航前需先设置 Zustand store
2. **"查看"/"查看详情"按钮选择器类型** → `<Button type="link">` 不是 `<a>` 标签，所有相关测试需修正选择器
3. **antd Table 行选择器** → 用 `tr.ant-table-row` 而非 `tr`
4. **toContainText strict mode** → 改用 `filter` + 单元素断言

### 需 PM 确认项

5. **车牌脱敏星号数量**：7 位车牌 `京A88862` → `京A***62`（动态星）还是 `京A**62`（固定2星）？

### 当前代码已正确但测试误判的项

6. **围栏车牌列占位符 `—`**：不是 Bug，测试断言需加上白名单
7. **蒙版 Modal 断言**：正则应匹配完整 17 位格式
8. **VIN 脱敏已实现**：所有页面代码均已添加 `maskVin()`/`maskPlate()` 调用，测试失败均因选择器不匹配
