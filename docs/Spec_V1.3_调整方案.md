# Spec V1.3 - PRD 对标调整方案

> 基于 PRD v1.3（2026-06-16）与当前代码库对比分析，梳理需要调整的功能、字段和逻辑。

---

## 一、类型系统变更（src/types/index.ts）

### 1.1 FaultType 枚举扩展（⚠️ 重要）

**现状：** 仅 4 种故障类型
```ts
export type FaultType4 = 'VDC' | 'CDCU' | 'BDCU' | 'ADAS';
```

**PRD 要求：** 9 种故障类型
| 枚举值 | 中文名称 |
|---|---|
| VDC | VDC故障报警 |
| CDCU | CDCU故障报警 |
| BDCU | BDCU故障报警 |
| ADAS | ADAS故障报警 |
| DC_DC_TEMP | DC-DC 温度报警 |
| DC_DC_STATUS | DC-DC 状态报警 |
| MOTOR_CTRL_TEMP | 驱动电机控制器温度报警 |
| MOTOR_TEMP | 驱动电机温度报警 |
| HV_INTERLOCK | 高压互锁状态报警 |

**调整：** 重命名 `FaultType4` → `FaultType`，扩展为 9 个枚举值。

### 1.2 BatteryAlert 类型枚举扩展（⚠️ 重要）

**现状：** 仅 5 种
```ts
type: 'SOC过低' | '电池高温' | 'SOC跳变' | '充电故障' | '温差报警';
```

**PRD 要求：** 14 种电池报警类型
| 枚举值 |
|---|
| 温度差异报警 |
| 电池高温报警 |
| 车载储能装置过压报警 |
| 车载储能装置欠压报警 |
| SOC低报警 |
| 单体电池过压报警 |
| 单体电池欠压报警 |
| SOC 过高报警 |
| SOC 跳变报警 |
| 可充电储能系统不匹配报警 |
| 电池单体一致性差报警 |
| 车载储能装置类型过充 |
| 绝缘报警 |
| 充电故障报警 |

**调整：** 扩展 `BatteryAlert.type` 为 14 个枚举值。

### 1.3 DrivingAlert 类型变更（⚠️ 重大变更）

**现状：** 传统驾驶行为分类
```ts
type: string; // 'Rapid Accel', 'Hard Brake', 'Sharp Turn', 'Fatigue', 'AEB'
```

**PRD 要求：** 对车/对人分类体系
| 枚举值 | 中文 |
|---|---|
| FCW1 | 对车一级预报警 |
| FCW2 | 对车二级紧急报警 |
| AEB_V | 对车AEB制动 |
| PCW1 | 对人一级预报警 |
| PCW2 | 对人二级紧急报警 |
| AEB_P | 对人AEB制动 |

**调整：** 重定义 `DrivingAlert.type` 枚举值。i18n 中已有对应 key（`alert.FCW1` 等），但 mock 数据和页面渲染均需同步修改。

### 1.4 Vehicle 类型补充字段

**现状：** Vehicle 缺少以下展示字段：
- `locationText`：最后位置的逆地址解析文字（当前用 chileAddresses 数组模拟）

**调整：** Vehicle 类型新增可选字段 `locationText?: string`，不再在列表页用 index 映射假地址。

### 1.5 BatteryMonitorItem 补充字段

**现状：** 缺少「日均电耗」和「充电状态」枚举
```ts
status: 'charging' | 'idle' | 'unknown';
```

**PRD 要求：** 充电状态枚举扩展为：
- `未充电` / `准备充电` / `充电中` / `对方放电` / `故障&异常`

**调整：**
- `BatteryMonitorItem` 新增 `dailyConsumption: number`（日均电耗 kWh/100km）
- `status` 枚举改为 5 个值

### 1.6 ChargeRecord / DischargeRecord 补充

**PRD 要求：** 充放电记录不包含 `count` 字段，充电/放电判定逻辑由后端信号判断。

**调整：** 移除 `ChargeRecord.count` 和 `DischargeRecord.count` 字段（如后端不需要）。

### 1.7 TripDetail 补充字段

**现状：** TripDetail 已有 alertCount/alerts，但 PRD 定义预警次数包含「围栏预警、故障预警、电池预警、驾驶预警」四类。

**调整：** TripDetail 类型保持不变，但预警次数计算需汇总四类预警。

## 二、全局逻辑变更

### 2.1 智利时区处理（⚠️ 新增）

**PRD 要求：**
- 冬令时 UTC-4：每年 4 月第一个星期六 24:00 → 9 月第一个星期六 24:00
- 夏令时 UTC-3：每年 9 月第一个星期六 24:00 → 次年 4 月第一个星期六 24:00

**现状：** 使用浏览器本地时区，未做智利时区特殊处理。

**调整：**
- `src/utils/format.ts` 新增 `toChileTime(date)` 工具函数，自动判断冬令时/夏令时
- 全局时间展示统一使用该函数转换
- 涉及文件：所有展示时间的页面组件

### 2.2 VIN 脱敏（✅ 已实现）

当前 `maskVin()` 已按"前6 + 7个* + 后4"实现，与 PRD 一致。无需调整。

### 2.3 车牌号脱敏（✅ 已实现）

当前 `maskPlate()` 已按"首尾各2 + 中间*"实现，边界情况（≤4位不脱敏）也已处理。无需调整。

### 2.4 地理位置精度（✅ 已实现）

当前 `truncateLocation()` + `LocationPrivacy` 组件已实现街道级精度。无需调整。

### 2.5 检索匹配规则（✅ 已实现）

当前 `matchVinSearch()` / `matchPlateSearch()` 已支持明文+脱敏匹配。无需调整。

### 2.6 数据范围定义（⚠️ 需完善）

**PRD 要求：**
- 当前租户数据隔离：所有模块仅展示 vehicle.tenant = currentTenant 的数据
- 例外：业务管理、实时监控菜单展示全部下级租户数据

**现状：** 当前 mock 数据未实现租户隔离逻辑，所有数据平铺展示。

**调整：**
- 在各页面的数据获取函数中增加 `tenantId` 参数过滤
- Dashboard/Vehicles/Risk/Driving/Battery/Trips/Fence/Repair：仅展示当前租户数据
- Monitor/Biz：展示全部下级租户数据
- **注意：** 这是后端逻辑，前端需做好 tenantId 参数传递。当前 mock 阶段可先按固定 tenant 过滤。

- 将 i18n `fence.radius` 改为 "半径(km)"

**PRD 要求：**
- 围栏关闭（未生效）时才支持编辑

**现状：** 编辑和删除按钮已实现 disabled 逻辑（status === '生效中' 时 disabled），二次确认也已实现。✅ 基本一致。

### 5.3 自定义围栏点位上限

**PRD 要求：** 最多 99 个点位

**现状：** 无点位上限校验。

### 5.4 围栏操作人/操作时间

## 八、电池管理（Battery）变更

### 8.1 电池监控 - 充电状态枚举扩展

**现状：** charging/i

dle/unknown（3 种）

**PRD 要求：** 5 种
- 未充电（对应信号值 0000b）
- 准备充电（对应信号值 0001b）
- 充电中（对应信号值 0100b/0011b）
- 对方放电（对应信号值 0101b）
- 故障&异常（对应信号值 0110b）

**调整：** 
- 类型定义扩展为 5 个枚举
- mock 数据和页面渲染同步更新
- i18n 新增对应翻译

### 8.2 电池监控 - 日均电耗字段

**PRD 计算公式：**
```
日均电耗(kWh/100km) = (当日0时累计电耗 - 昨日0时累计电耗) ÷ (当日0时累计里程 - 昨日0时累计里程) × 100
里程为 0 时，电耗显示为 0
```

**现状：** 无此字段。

**调整：**
- `BatteryMonitorItem` 新增 `dailyConsumption: number`
- 列表新增「日均电耗」列
- 详情页「日均电耗趋势」折线图展示近30天数据

### 8.3 充放电记录 - 判定逻辑文档化

**PRD 充电判定：** 车辆充电状态信号 = 0100b/0011b 且电流 > +2A 且持续 1 分钟
**PRD 放电判定：** 车辆充电状态信号 = 0000b 且电流 < -2A 且车速 > 0 且持续 1 分钟

**现状：** 这是后端数据加工逻辑，前端无需实现。但记录时间格式需统一为 YYYY-MM-DD HH:mm:ss。

### 8.4 充放电记录 Tab 标题

**PRD 导航：** 电池管理 → 充放电记录（充电记录 + 放电记录两个子 tab）

**现状：** 二级 tab 为「充电记录」「放电记录」。✅ 与 PRD 一致。
