# 自动化测试与 Bug 排查报告

### 1. 测试概述
- **测试地址**：本地环境 `http://localhost:3000/`
- **审查范围**：根据 PRD 及真实页面交互排查，对当前项目源码及功能逻辑进行了全面复核。确认了您指出的所有问题在当前代码中**均真实存在，确系未修复的 Bug 或未实现的需求**。

### 2. 存在的 Bug 与问题列表（待修复）

#### 1) 首页看板菜单
- **地图布局不合理**：首页看板的地图尺寸/高度太小，版面局促，需要调大占比。
- **数据指标样式与文案问题**：“今日里程”和“累计总里程”的单位（公里数字符）字号过大，没有与数值做明显的视觉区分，且文案需要修正。
- **排行榜数据缺失**：今日预警排行榜中，各预警类型的具体数据列为空白（由于前端组件取值的 key 与 Mock 数据返回的 key 不一致导致无法渲染）。

#### 2) 车辆管理菜单
- **位置信息未解析**：列表中的“位置”数据有问题，当前仅显示城市名。需求要求必须是准确的地理位置信息（如示例：`智利圣地亚哥首都大区圣地亚哥市阿乌马达步行街234号`）。
- **车牌号格式不符**：车辆的车牌号仍为测试用的旧格式（如 `WDLV-xx`），未修改为符合要求的格式（如示例：`KLTX56`）。

#### 3) 风控预警菜单
- **出栏报警数据空白**：围栏报警菜单中，报警类型对应的数据映射有问题，导致出栏报警等部分类型显示为空白。
- **位置信息未逆解析**：页面列表中的位置信息展示为数字经纬度坐标，需求要求必须展示准确的文本地理位置信息。
- **故障报警表头及内容错误**：列表表头叫成了“内容”而不是“报警内容”，并且列表中具体的报警内容字段值也与 PRD 中的规范要求不对应。
- **中文化翻译严重遗漏**：故障报警、电池报警的操作列表字段（如表头显示 `common.action`）、列表内的操作按钮、以及电池报警的“报警类型”统统是英文代码或英文单词，未作中文处理。

#### 4) 驾驶行为菜单
- **位置信息未逆解析**：驾驶预警列表中的位置信息同样显示的是经纬度，而非要求的准确地理位置信息。
- **月报/周报切换无效（假联动）**：在驾驶报告菜单中，点击“月报”时，列表里的周期列并没有切换成月报维度的数据，两者没有做出功能差异，只是空壳切换。

#### 5) 电池管理菜单
- **充电状态及操作列未中文化**：电池监控菜单中的“充电状态”字段显示为英文（如 charging / idle），操作列表头的字段值也是英文。
- **顶部统计数据未中文化**：页面顶部的数据指标中，第 4 个指标统计卡片的内容没有做多语言适配，显示为英文。

#### 6) 围栏管理菜单
- **新建围栏无法选点**：在创建/新建围栏的弹窗中，虽然渲染了地图，但地图组件未绑定任何点击交互事件，导致用户无法在地图上选点来获取坐标。
