# 车辆信号数据与导出功能 PRD

- 新增一级菜单车辆数据，二级菜单车辆信号数据、数据导出记录
- 此菜单排序在一级菜单“业务管理”后面

## 1. 车辆信号数据

### 1.1 页面布局结构 (Layout Breakdown)

- **Layout模式**：左右分栏（Left Sidebar + Right Main Content）。
- **Left Sidebar**：车辆组织架构树 (`VehicleTreeComponent`)，固定宽度，内部支持垂直滚动。
- **Right Main**：分为上部筛选项区域 (`FilterBarComponent`)、中部数据表格区域 (`DataGridComponent`)、下部分页与操作区域 (`PaginationComponent`)。

### 1.2 核心组件与交互逻辑 (Components & Interactions)

### A. 左侧车辆选择树 (`VehicleTreeComponent`)

- **数据结构**：多层级树状结构（大区 -> 车队 -> 车辆）。仅叶子节点（车辆）可被勾选。
- **状态展示**：顶部标题需实时显示选中数量 `车辆选择 ({selectedCount}/10)`。
- **交互规则**：
    - 支持通过搜索框对 VIN/车牌号进行模糊匹配过滤。
    - **限制条件 (Constraint)**：最多允许勾选 10 辆车。当尝试勾选第 11 辆时，需拦截并阻断状态更新，可触发轻提示 (Toast)。

### B. 顶部筛选栏 (`FilterBarComponent`)

- **上报时间选择器**：
    - 类型：Datetime Range Picker。
    - 格式：`YYYY/MM/DD HH:mm`。
- **信号选择器**：
    - 类型：多选下拉菜单 (Multi-Select Dropdown)。
    - 文案状态：未选择时显示占位符，选中后显示 `已选择 {n} 项信号`。
- **操作按钮**：
    - **查询 (Query)**：校验表单（必须选车、选时间、选信号），校验通过后触发网络请求，更新下方表格数据。
    - **重置 (Reset)**：清空时间、信号选择状态，重置表格为空状态。

### C. 数据表格 (`DataGridComponent`)

- **表头生成逻辑 (Dynamic Columns)**：
    - 固定列：`序号` (Index), `上报时间` (Timestamp, 需支持点击排序), `设备ID` (Device ID)。
    - 动态列：根据“信号选择器”选中的项动态生成列（如 SOC, SOH, 总电压等）。
- **数据格式化字典 (Formatter Rules)**：
    - **数值型 (Numeric)**：需拼接对应单位。例如：SOC -> `{value} %`, 总电压 -> `{value} V`, 总电流 -> `{value} A`, 最高单体温度 -> `{value} ℃`, 绝缘电阻 -> `{value} kΩ`。
    - **枚举/布尔型 (Enum/Boolean)**：需转义为可读文本。例如：温度差异报警 -> `正常` / `异常`，充电状态 -> `充电完成` / `停车充电` / `未充电` / `行车充电`。
    - **空值回退 (Fallback)**：字段缺失或为 null 时，统一渲染为半角破折号 `—`。

### D. 分页与导出区 (`PaginationComponent`)

- **展示**：左侧展示总条数 `共 {total} 条记录` 和页大小切换器 `每页 [100] 条`。右侧为分页页码器。
- **导出按钮 (Export Action)**：
    - 点击“导出 Excel”按钮，读取当前有效的查询条件和表格结果。
    - 触发异步导出任务生成，成功后提示用户前往“数据导出记录”查看，并静默向导出记录列表中增加一条状态为“处理中”的数据。

## 2. 数据导出记录

### 2.1 页面布局与文案

- **顶部提示**：主标题“导出记录”，副标题“最多保留 30 天内的记录，文件有效期 7 天。”。
- **主体区**：占据全屏宽度的列表表格。

### 2.2 数据表格列定义 (Table Schema)

| **列名 (Column)** | **数据源/格式化规则 (Data Mapping)** | **示例渲染 (Example)** |
| --- | --- | --- |
| **文件名** | Excel 图标 + 自动生成的文件命名规则 | `[Icon] 车辆信号数据_VIN000..._20...` |
| **筛选条件摘要** | 提取创建导出时的核心筛选条件拼接字符串 | `多车(2辆) | 1小时 | 8个信号` 或 `VIN: LSX123... | 选定时间 | 选定信号` |
| **数据条数** | 导出的总数据行数，需千分位格式化。若未计算完成显示 `-` | `27,567` 或 `-` |
| **创建时间** | 格式 `YYYY-MM-DD HH:mm:ss` | `2026-06-10 16:31:33` |
| **状态** | 枚举映射 UI 组件 | 见 2.3 状态流转 |
| **操作** | 基于当前状态渲染不同 Action | 见 2.3 状态流转 |

### 2.3 状态流转与渲染逻辑 (State Machine)

- **状态 `processing` (处理中)**：
    - 状态列UI：蓝色 Loading 圈 (Spinner) + 文案“处理中”。
    - 操作列UI：灰色失效文本“处理中”及 Loading 圈，不可点击。
- **状态 `completed` (已完成)**：
    - 状态列UI：绿色 Check 圆圈 + 绿色底色 Tag + 文案“已完成”。
    - 操作列UI：蓝色可点击的“下载”按钮带下箭头 Icon。点击触发文件下载动作。
